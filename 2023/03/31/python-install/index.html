linux环境下无网络安装python的方法

发表于 2023-03-31 分类于 python基础 阅读次数：

python安装包的下载

python环境的安装可以直接用anaconda,不过anaconda过于“沉重”,它自带的包管理器conda虽然方便，但也有限制较多、升级滞后和国内镜像少的问题。

如果要直接使用安装python和使用pip作为包管理器，可以从官网下载安装包，不过官网只提供了windows平台和mac平台的安装包，unix/linux平台提供了源代码，需要自己编译安装。虽然源码安装实际也很简单和方便，但是对于非联网环境来说，下载编译环境需要的各种库和头文件是件麻烦事，直接使用针对不同linux版本的二进制包下载后，在断网环境下安装是最方便的方法。 为了避免麻烦，最简单的方式是配置一个和非联网环境完全相同的linux环境，用虚拟机或docker都可以，然后在联网环境用apt-get download下载python安装包，复制到非联网环境安装，这样在后续需要下载python软件包的时候，也可以在这个环境用pip download下载，不会出现版本差异无法安装的问题。

python软件包的下载

后期的python软件包可以同样在这个环境下用pip download XXXX下载后，复制安装，这个命令下载的时候默认会把所有的依赖都下载下来，这样复制安装就很方便，如果不需要下载依赖，就用pip download XXXX --no-deps

linux环境下其他依赖软件的安装

如果部分python软件包还需要linux的软件包，可以仍然用sudo apt-get download XXXX下载，和python软件包的pip download不同的是，这个命令下载的时候是不会下载依赖包的，不过可以用sudo apt-cache depends XXXX|grep "Depends\|Suggests"|cut -d: -f2 获取某个软件包的全部依赖。如果存在依赖的依赖，那就只有写脚本用递归方式获取了，具体可以参考 https://www.jianshu.com/p/434a50c6c0ed
