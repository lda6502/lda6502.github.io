使用github发布个人博客

我们在本机搭建好个人博客网站后，如果希望别人也能看到，就需要发布到网上，这里我们使用github来发布我们的博客内容，需要做以下几个方面的工作

一、注册github账号，配置访问密钥

在使用github发布博客内容之前，我们需要首先注册github的账户，并且设置好登录github的登录密钥等。github现在已经不支持简单的账户、密码的登录方式了，需要采用密钥方式才能访问，主要需要做的工作是：

  1. 通过ssh-keygen生成ssh访问的公钥和私钥
  2. 登录github网站，在相应的配置页面将公钥粘贴复制到你的账户上
  3. 配置私钥用于ssh登录，其实就是将私钥复制到ssh登录的配置目录

以上操作的具体步骤和说明，网上较多，这里就不细说了。如果上面的操作都正确的话，就可以测试登录是否正常了，在终端窗口输入以下命令：

ssh -T git@github.com

如果显示类似内容，那就表示密钥方式访问设置成功

Hi XXXXXX! You’ve successfully authenticated, but GitHub does not provide shell access.

显示的内容中，那个XXXXXX应该是你自己的github账户名。

二、生成和配置hexo访问github的令牌

  1. 点击github网页右上角的个人配置图标，在下拉菜单中选择Settings，就是下图红框标注的那个位置

  1. 在最左边的网页选项中（在最下面一项）点击 Developer settings，就是下图红框标注的位置

  1. 在进入的页面中，选择最左边的 Personal access tokens 选项，然后点击右上角的 Generate new token 选项，就是下图红框标注的2个地方

  1. 在token生成的页面输入必要的选项，然后点击最下面的 Generate token 按钮，需要输入的选项主要有3个，第一个是token的备注，随便输入就可以，第二个是token的有效期，默认是30天，就是说30天后还要重新生成和配置，可以把这个时间设置长一点，甚至设置为永久生效，当然安全性要受点影响，第3个是勾选token的使用范围，勾选第一个 repo 就可以了，具体见下图红框标注的3个地方

  1. 最后复制生成的token，就是下图红框覆盖的部分，hexo用这个token就可以把我们本地的博客发布到github了

三、新建github仓库，配置hexo并发布内容

  1. 在github上新建1个仓库，仓库名称是 XXXXXX.github.io ,这里XXXXXX是你自己的github账户名

  2. 安装hexo的一键发布github软件包，安装命令是 npm install hexo-deployer-git --save

  3. 打开blog下的 _config.yml 文件，找的deploy部分，按照下图的内容配置发布方式和仓库的url，其中红色覆盖的部分就是刚才生成的token，两块绿色覆盖的都是你的github账户名

  1. 使用 hexo g 命令生成博客内容 ，在用 hexo d 命令发布到github上，如果发布成功，过十几秒钟，你就可以用 XXXXXX.github.io 的网址访问你的博客了，以后如果你有新的博客内容，要更新发布也是依次用这两个命令就可以了

需要注意的是，现在github在国内访问有的时候不太稳定，以上过程如果有连接方面的问题，可能就需要自己找下国内访问比较稳定的方式了。

最后还有一点关于图片的说明，如果有图片的话，需要在source目录下建目录，例如：image，然后将图片都放在这个目录中，在博客内容中，可以用md格式引用图片，可以用 image/图片文件名 的方式引用,例如: image/hexo_config.png

个人博客的使用网站的使用

  • 我们使用hexo完成博客网站的搭建后，就可以编写新的博客了，其实在我们完成博客网站的搭建后，通过浏览器已经可以看到我们的默认的博客网站和一篇hello world的博客，在正式使用前，我们可以做一点必要的设置，包括我们博客网站的名字和描述之类的，在我们的blog目录中，有个 _config.yml 文件，用文本编辑器修改这个文件的内容，可以对我们的网站进行配置，至少需要对 title和language 这两项设置进行修改，其他的内容如果不了解可以先保持不动，title是设置我们网站的名称，language是设置网站的语言，例如我们的内容是中文，就设置为 zh-CN 。

  • 网站的基本设置完成后，我们就可以新建博客，用命令 hexo init hello ，就可以新建文件名为hello.md的博客，进入blog/source/_posts 目录中，我们发现除了初始化的时候自动生成的的 hello world的的博客外，新生成了一个 hello.md 的文件，用文本编辑器打开这个文件，可以发现类似如下的内容：

---
title: hello
date: 2022-05-14 12:09:19
tags:
---

  • 修改title的内容，就可以编辑博客的标题，其他的部分如果不了解，可以暂时先不动，直接在文本最后开始书写我们自己的博客内容了，博客内容是md格式，如果你没有特殊的格式要求，就直接写成纯文件文件就可以了。如果有兴趣了解md文本的格式设置方法，也可以简单学习点常用的设置方法，具体可以看这个网站 http://markdown.com.cn ，博客写好后，保存后，在网页上刷新一下，就可以看到我们刚才写的博客内容了 ，按照这个方法，我们就可以继续写新的博客了。

  • 删除博客 ，找到 blog/source/_posts 目录中，需要删除的那个博客文件，直接删除这个文件就可以了，刷新网页我们就可以看到这个博客已经删除了。

  • 修改博客 ，同上面找到对应的博客文件，继续编辑这个文件的内容，就可以进行内容的修改了。

好了，现在开始我们就可以愉快的写我们自己的博客，并且在我们本机的浏览器上查看了。

hexo的简单介绍和使用

简单来说，我们可以把hexo看成一个md文件的转换和发布工具，就是说我们只需要用markdown写一系列的文档，hexo可以自动帮我们转换为一个静态的网站的内容，hexo同时还提供发布功能，就是在我们本机启动web服务器，让我们可以通过浏览器直接参看我们的博客效果，当然也支持一键发布到github上，这样其他人也可以看到我们的博客内容了。

所以简单的来说，hexo的基本功能是转换md格式的文档，这样就省去了我们写html网页的麻烦，另外在本机实现了一个web服务器的功能，这样即使不上网，我们也可以在本机浏览器查看我们的博客内容了。

hexo并不提供md文件编写相关的内容，md文件的编写我们可以找自己熟悉的文本编辑器或者其他的专用编辑器。

hexo是基于nodejs开发的，所以在安装和使用hexo之前，你需要安装nodejs，你可以到官方网站下载和安装 https://nodejs.org ,nodejs安装后，我们需要知道 npm 命令。

npm ,命令是nodejs的软件包管理工具，就是说基于nodejs的软件包的安装、升级和删除等处理都要用npm来完成，那啥是软件包呢？实现某种功能的完整的程序，我们把它封装在一起，就是一个软件包，例如hexo就是一个软件包。

使用npm命令，就可以解决我们安装和使用hexo的需求了。下面是hexo安装和初始化的步骤：

  1. 安装hexo软件包，npm install hexo-cli -g ,注意这个命令在linux和mac下执行的时候可能需要root权限，就是说这个命令要么用sudo执行，要么用root用户执行。hexo安装完成后，我们就可以用直接用hexo这个命令完成我们需要的功能了。
  2. 创建和初始化博客目录 ，假如我们希望创建的博客目录是当前目录下，博客的目录名为blog，那创建和初始化的命令就是 hexo init blog，当然如果完全可以使用其他的目录名。
  3. 进入目录，如果我们的目录名为blog，就可以用cd blog进入已经初始化的目录中
  4. 安装hexo需要用到的其他软件，npm install
  5. 启动web服务，hexo server ,web服务会在默认的4000端口启动，在浏览器输入 http://localhost:4000/ 就可以看到我们的博客页面了。

关于个人博客网站的搭建

个人博客可以记录自己学习、工作和生活中的想法、备忘和需要总结的内容，无论从哪个角度来说，都是很有意义，也很有价值的，当然可能会花点点时间，但是长远来看，这点代价（这个看你看问题的角度了，可以认为是代价，也可以认为是学习和个人价值提升的过程）是值得的。
如果有写博客的想法，那剩下的问题就是用啥工具来写？写了之后放在哪里？
如果你不熟悉软件和系统配置，而且也希望尽可能多的人能看到自己写的内容，那就找个博客平台，注册个账号，就在平台上写就行了。使用博客平台，好处是方便，不好的地方是个性化和自主性方面不足。如果你熟悉软件和系统配置，那完全可以搭建个自己个人的博客网站，而且非常的简单和方便，并不会增加多少工作量，下面我们主要就讨论下这个问题。
个人博客其实简单的说有个web服务器，把写的内容输出为html格式的网页就可以了。但是先不说大部分人并不熟悉html，就算是非常熟悉，每篇文档都写成html还是太麻烦了一点，还有就是即使是配置个简单的web服务器，也还是有不少的安装、配置和维护的工作量，实际上我们可以用一些简单的办法解决这两个问题。

  • 关于内容编辑，我们可以直接用hexo这个简单的博客框架，它主要的好处是我们可以直接用markdown格式书写博客，这个框架自动把这些内容转换为静态的html网页，这个过程是自动完成的，而且markdown的使用非常简单，我们用最简单的文本编辑软件就可以编辑md格式的博客内容，当然你要是愿意，也可以找到专门编辑md格式的软件。
  • 如果我们只是在本机自己用看的话，那用hexo启动博客服务就可以了，如果要发布到网上让其他人也能看到，简单的话可以直接放到github上，hexo提供简单的命令，就可以把本机的内容发布到github上，当然如果你想进一步建立自己的服务器，也是可以的。

下面是以上说到的工具的相关网址：

  • hexo的官方网站,(https://hexo.io)
  • github提供的网站托管空间,(https://github.io)
  • markdown的教程,(https://markdown.com.cn)

后面我们会继续介绍hexo的具体使用方法。
