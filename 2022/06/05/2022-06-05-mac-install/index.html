MacOS的降版本安装

发表于 2022-06-05 更新于 2022-06-06 分类于 MacOS基础 阅读次数：

苹果电脑的MacOS操作系统需要重装的机会很少，正常情况下电脑的使用周期范围内可能都不需用任何的系统重装，只要正常按照升级提示进行安全升级或者新版本升级就可以了。但是一般来说，新的系统对硬件配置要求更高，有时候升级后才发现，自己老的硬件配置带不动新系统的时候，通常就没有后悔药可以吃了，因为苹果系统一般来说并没有给你直接的降级通道，甚至新版本出来后，你在应用商店中都有可能找不到可以下载的老版系统了，这篇文章主要就是讲讲如何降级的问题，当然对于早期的SSD硬盘没有焊死的苹果电脑，如果你需要升级下硬盘，那这篇文章可能也能帮到你。

  • 如何创建可引导的 macOS 安装器

关于数据备份的问题

  • 系统自带的备份功能
    重装系统原来的数据会全部覆盖，所以在这之前你需要备份数据，由于我们重装前后是两种不同版本的操作系统，所以简单的用系统自带的备份功能有可能是不行的（理论上低版本的系统可能无法完全向上兼容），但是用系统自带的功能做下备份还是很有必要的，因为实在不行可以还原到原来版本的系统，所有的程序和数据都还是可以恢复的。

  • 手工备份
    如果我们对自己需要备份的数据比较清楚，就可以手动把需要的软件和数据备份一下了，就是只有用copy的方式备份相关文件夹的数据。用访达打开你的硬盘，通常有系统、应用程序、用户、资源库4个文件夹，如果搞不清楚，就找个大点的移动硬盘，都备份一下吧。当然如果对自己的系统比较了解的话，选择需要备份的文件夹就行了。用访达打开硬盘看到的4个文件夹，实际对应文件系统根目录的几个文件夹，我们稍微说明一下。

  • 系统
    这个文件夹对应根目录中的System目录，主要是系统和配置相关的文件。

  • 应用程序

    • 这个文件夹对应根目录中的Applications目录，所有的应用程序，包括系统自带的和用户自己安装的都在这个文件夹中，在终端端口，用ls -l /Applications命令可以看到，其实每个应用程序就是一个目录，这个目录的名字就是应用程序的英文名，后缀都是.app，所以本质上在MacOS中每个应用程序就是用这个程序的英文名加后缀.app的目录。
    • 所有的程序相关的文件都在这个目录中，例如Safari浏览器的目录名就是Safari.app，这个目录中的内容都遵循苹果的规范，包括这个程序在图形界面下显示的图标，和用户点击时实际运行的程序文件。在访达或者启动台中查看应用程序时，实际显示的就是按照苹果的规范存在程序目录中的图标文件，你点击运行程序时，系统实际就是按照规范运行对应目录中的某个程序文件。
    • 所以MacOS中你在访达或者启动台中把这个程序拖到废纸娄中删除程序的操作，其实就相当于删除了这个目录。所以大部分情况下，MacOS中安装应用程序实际就是把程序文件夹复制到这个目录中。当然部分程序在安装后可能还有点配置数据会放在系统文件夹中。

  • 用户
    这个文件夹对应根目录中的Users目录,下面有每个用户有个独立的目录，中文用户的目录名用的是用户名的拼音，用户的资料都放在自己的目录中，对应有文档、下载、桌面等子目录，同样道理，在图形界面显示的是中文，在文件系统中实际的目录名是英文Document、Downloads、Desktop。

  • 资源
    这个文件夹对应根目录中的Library，顾名思义就是系统或者应用程序需要用到的一些数据或者配置文件。

关于系统安装包的下载

苹果在系统升级后，老版系统的安装包在应用商店中就搜索不到了，所以如果我们要回到老版系统，首先需要下载对应的安装包。虽然应用商店搜索不到了，但是实际还是存在的，可以按照前面的官方说明链接查看下载。按照官方的说明，macOS Sierra 及之前版本是dmg文件，之后的版本就都是前面说的.app为后缀的安装包了。

制作可以启动的U盘安装盘

这种方式可以不用联网，而且做好的安装盘可以重复安装多台机器，具体参考前面官方说明

用U盘安装系统

将做好的启动盘插入电脑，按住option键开机（pc键盘对应的是alt键），看到出现可引导磁盘的选项时，就可以松开按键，选择U盘安装了。

恢复备份的数据

安装完成后，就可以把手动备份的数据恢复，重新安装需要的应用了。当然也可以尝试用系统备份的数据恢复，如果两个版本的差异不大，有可能可以正常恢复，当然这个过程如果有问题也没关系，就用手工备份的数据好了。
